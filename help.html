What is this chart showing ?
This is a sample of real estate sales in the city of Strasbourg in 2012. Each buble represents an appartement or a house.
It is just an example thrown together for this experiment - you could easily imagine other applications for such an interactive visualization, where the bubles would be countries, people or projects.
What can I do with it ?
  • Re-position the bubles across various dimensions: price, surface, district, etc... You can pick one or two of them, and based on the type of dimensions you picked the chart will adapt itself.
  • Select how the bubles are colored.
  • Select how the bubles are sized.
  • For the selected dimensions, select the visualization type. It's usually list or a chart, but there could be several chart types in some cases.
  • Filter the bubles.
What about these strange axis ?
In some charts the axis are used to show some statistical data. Inspired from E. Tufte.
What's the purpose of this thing ?
It's an experiment around the concept of object constancy. Object constancy ? I'll just say that it is about keeping things visually consistent and smoothly transionned while changing a chart's parameters (filters, analytical dimensions...). For a much better explanation, see this paper from Jeffrey Heer and George Robertson.

d3.js is a great way to achieve object constancy, with its enter(), update() and exit() pattern for mapping data to visual markers and its support for transitions. Look at this impressive demo and these blog posts (this and that) from Mike Bostock (d3's creator). While d3.js allows you to do such things with relatively little code, you will still have to develop a custom function for transitionning between 2 types of chart. It therefore doesn't scale well when the number of possibilities increases. In the approach I'm trying here, the constancy is ensured by the bubles: whatever chart type you select, the bubles will always be there - what changes is how they get positionned and some additionnal chart components such as axis, background bars or boxplots. It makes it much easier to develop since regardless of the combination of chart types involved in the transition, the same functions are used to move, color and size the bubles. The old additionnal components are simply removed and the new ones are added. A shortcoming of this is that it's not very suitable for aggregating quantitative values, for example total cost by district. But you can still do it for example by piling the bubles and make them a bit more squared to form a bar shape, or simply by drawing a shape in the background behind the bubles (to see an example, plot by district and price and select bars in the top right radio buttons).

Is the whole thing really useful ? I'm not sure, but it's interesting and was fun to do :-)
The next step (on a difefrent topic) would be to rework the user interface for changing the chart parameters and make it a commandline interface: "plot by district and price, color by surface". The challenge will be to help the user type the query properly in a semi-formal language - we need something better than auto-complete here.
